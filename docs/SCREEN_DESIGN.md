# 黒猫飯店 画面設計ドキュメント（SCREEN_DESIGN）

本書は「黒猫飯店」の画面構成と画面遷移の設計をまとめる。
タスクバーヒーロー（TBH）の画面設計を下敷きにしつつ、ポモドーロタイマー＋経営シムという独自要素を統合する。

---

## 1. 設計思想

### 1.1 タスクバーヒーロー（TBH）からの借用
TBH は「放置 × ハクスラ × 編成」をシンプルなタブ構成に落とし込んでいる。本作はその情報設計を参考にする。

| TBH の概念 | 役割 | 借用ポイント |
|---|---|---|
| **Cube（キューブ）** | 装備の加工・強化を集約した工房 | 装備操作を1か所に集約し、進行度で段階解放する構造をそのまま採用 |
| **Formation（編成）** | パーティ／配置の編成 | 扉方針・店番・献立デッキの「編成」に対応 |
| **Stash（倉庫）** | アイテム保管 | 独立タブをやめ、工房のサブビューに統合 |
| **Rune Tree（ルーンツリー）** | 永続強化ツリー | 「内装改装ツリー」に対応 |

### 1.2 独自要素との統合方針
- **ポモドーロタイマー**：集中＝ダイブ（探索配信）の駆動源。`▶集中` が探索のトリガになる。
- **経営シム**：店の営業・献立・店番が日次サイクルで回る。探索の戦果が店の経営に還元される。
- 上記2軸を **ホーム（店）** で束ね、探索（ダイブ）→ 帰還（BagModal）→ 翌日営業、という1日のループに落とす。
- 装備まわりの複雑さはすべて **工房（Cube）** に隔離し、コアループ画面はシンプルに保つ。

## 3. 工房（Cube）詳細設計

TBH のキューブシステムを参考に、進行度（探索フロア）で段階解放する。

| Lv | 機能 | 解放条件 | 内容 |
|---|---|---|---|
| Lv1 | **分解** | 初期から | 装備 → 廃材 |
| Lv2 | **合成** | B3 クリア | 同グレード5個 → 1ランク上（10% 大成功で2ランク上） |
| Lv3 | **刻印** | B5 クリア | 廃材50 でアフィックスをリロール |
| Lv4 | **装飾（ソケット）** | B8 クリア | 英雄以上に宝石をはめる。部位別に効果が変わる |
| Lv5 | **精錬** | B12 クリア | 同グレード装備を餌にしてアフィックス値を底上げ |
| — | **除去** | Lv3 と同時 | 装飾・刻印を外して付け替え |

### 3.1 倉庫ビューの統合
旧「倉庫タブ」は廃止し、**工房のサブビュー** として統合する。
工房に入ると各加工機能（分解／合成／刻印／装飾／精錬）と倉庫ビューが同一画面内で切り替え可能になる。
これにより「倉庫で装備を選ぶ → 工房で加工する」という画面往復を1画面に閉じる。

## 4. 画面遷移フロー

### 4.1 コアループ（1日のサイクル）
```
ホーム ──▶集中──▶ ダイブ ──完走/浮上──▶ BagModal ──振り分け──▶ ホーム（翌日）
  ▲                                                                │
  └────────────────────────── 営業ライブ ◀──────────────────────────┘
```

### 4.2 ホームからのサブ画面遷移
```
ホーム ──⚙──▶ 編成・献立 ──←──▶ ホーム
ホーム ──📦──▶ 箱開封     ──←──▶ ホーム
ホーム ──🔮──▶ 改装       ──←──▶ ホーム
ホーム ──顔タップ──▶ 会話 or キャラ詳細 ──←──▶ ホーム
```

### 4.3 フッタータブ間（横移動）
```
店 ⇄ メンバー ⇄ 工房 ⇄ 市場 ⇄ 経営
```

### 4.4 キャラ詳細 → 工房の直通導線
```
キャラ詳細 ──装備タップ──▶ 倉庫候補表示 ──「工房へ」──▶ 工房（選択装備の強化画面）
```
装備枠をタップすると倉庫から装着候補が提示され、さらに「工房へ」を押すと、
その装備を選択した状態で工房の強化画面に直接遷移する。

## 5. TBH との対応表

| TBH | 黒猫飯店 | 備考 |
|---|---|---|
| **Formation（編成）** | 編成・献立（経営タブ） | 扉方針・店番・献立デッキの編成に対応 |
| **Cube（キューブ）** | 工房 | 装備加工を集約。進行度で段階解放 |
| **Stash（倉庫）** | 工房のサブビュー | 独立タブを廃し統合 |
| **Rune Tree（ルーン）** | 内装改装ツリー | 永続強化ツリーに対応 |
| **Portal（ポータル）** | ダイブ（探索配信） | `▶集中`＝ポモドーロが入口 |
| **Pet（ペット）** | キャラ（店番／探索メンバー） | キャラ詳細・好感度・会話で深掘り |
| **Skill（スキル）** | スキルツリー／スキル装備 | キャラ詳細モーダル内に格納 |

## 6. 未接続画面の救済方針

現在どこからも到達できない（または独立しすぎている）タブを、適切な親画面のサブビューへ畳み込む。

| 既存タブ | 救済先 |
|---|---|
| **記憶タブ** | メンバータブ内のサブビュー、またはキャラ詳細内 |
| **倉庫タブ** | 工房のサブビュー |
| **統計タブ** | 経営タブ内のサブビュー |
