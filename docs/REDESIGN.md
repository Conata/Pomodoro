# 黒猫飯店 — リデザイン提案（営業サイクルの是正 ＋ 画面刷新）

対象：`main.gd`（Godot 4.3 / モバイル一画面）。本書は **「なぜ変えるか」と「どこを触るか」** を
具体的に書く実装前の設計。既存 `DESIGN.md` / `WORLD.md` を上書きせず、差分提案として併読する。

---

## 1. 違和感の正体：「ポモドーロ後に閉店」問題

### 現状フロー（`enum Phase { MORNING, DIVE, CLOSE, NIGHT }`）

```
MORNING（開店前・準備） → DIVE（ポモドーロ＝潜行） → CLOSE（精算・閉店三行）
  → NIGHT（"閉店後"：箱開封・会話） → 翌 MORNING
```

集中（ポモドーロ）を完走した直後に、

- `CLOSE`：timer_label = **「精算」**、カード見出し = **「閉店三行」**、CTA = **「閉店作業へ」**
- `NIGHT`：timer_label = **「閉店後」**、パネル名 = **「閉店後」**

が出る。つまり **「頑張って集中 → 戻ってきたら店はもう閉まっていて、レシートを見せられる」**。
休憩（＝ご褒美の時間）がまるごと"店じまい後の事務処理"として描かれている。これがおかしい。

### あるべき姿（モックアップ ＋ `WORLD.md` の営業時間 22:00–4:00 と一致）

- **潜行中（集中）こそ店は閉まっている**：皆が電脳マンションに出払っている（時刻 ~18時台、開店前）。
- **ポモドーロ完走 ＝ 暖簾を出す＝店が開く**。休憩は **「黒猫飯店・営業中（22:30・賑わい）」**。
  仲間が戻り、客が来て、戦利品を整理し、会話し、ヒロインから次の依頼を受ける。**これが報酬**。

モックアップのサイクル（6ステップ）とも整合：
1 ポモドーロ開始 → 2 探索中 → 3 探索完了・結果回収 → **4 黒猫飯店で休憩（営業中）** →
5 依頼受注・準備 → 6 次のポモドーロへ。

## 2. サイクル是正：フェーズの「意味」と「ラベル」を貼り替える

enum の値は変えない（セーブ互換のため）。**意味とUI文言だけ**反転させる。

| Phase | 現状の扱い | 新しい扱い | 店の状態 |
|---|---|---|---|
| `MORNING` | 開店前・準備 | **仕込み（次の潜行の支度）** | 閉（開店前） |
| `DIVE` | ポモドーロ＝潜行 | 変更なし：潜行（集中） | **閉（皆が出払い）** |
| `CLOSE` | 精算・閉店三行 | **浮上・帰還（戦利品を持って戻る）** | 帰り道 |
| `NIGHT` | "閉店後" | **営業中（暖簾を出す＝休憩・団欒）** | **開** |

「閉店」という語は **潜行中＝店が閉まっている文脈以外では全部消す**。

### 触る箇所（`main.gd`、grep 済みの実在行）

- `_update_clock()`（L397〜）
  - `Phase.NIGHT`: `timer_label.text = "閉店後"` → **`"営業中"`**、status は「灯りがついて、皆が戻る。」系に。
  - `Phase.CLOSE`: `timer_label.text = "精算"` → **`"帰還"`**。
  - 可能なら NIGHT/CLOSE のタイトルに時刻感（22:30）を出す。
- `_build_night()`（L764）: `night_panel.name = "閉店後"` → **`"営業中"`**。
- `_build_close()`（L832, L836）: `_section("閉店三行")` → **`_section("今夜の首尾")`**、
  CTA `"閉店作業へ"` → **`"暖簾を出す"`**（＝店を開ける）。`close_text` の頭も「帰還報告」トーンに。
- `_apply_phase()`（L385〜392）: タブ可視ロジックはそのまま。タブ名（後述の刷新）に追従。
- オフライン文言 L98「【安息】閉店中に…」は **そのまま可**（離席＝本当に閉まっている）。
- `_on_close_done()`（L353）: NIGHT 入り口。挙動はそのまま、語だけ「営業開始」。

> 重要：`sim.close_day()` の **計算式（客数・単価・三行）は壊さない**。これは「今夜の営業の首尾」を
> 出すロジックとして再利用する。変えるのは**枠組み（閉店事務 → 開店して賑わう）**だけ。
> `DESIGN.md` の「朝の編成が夜の三行に返ってくる」という存在意義は維持される。

### セーブ互換

`enum` 値・`kuroneko-v3` キー・`close_day()` の戻り値スキーマは不変。**文言とビューのみ**の変更なので
既存セーブはそのまま動く。

## 3. 画面刷新（モックアップ準拠）

モバイル一画面・Control ベースを維持しつつ、ビジュアル言語をモックアップに寄せる。

### 3.0 共通フレーム
- **常設トップバー**：場所アイコン＋時刻＋所持金（¥表記）＋メニュー。
  例：`🌙 22:30  黒猫飯店・営業中    ¥23,450  ☰`／潜行時は `📡 18:22  電脳マンション 12F  …`
- パレット：ニアブラック地＋角丸カード＋琥珀/朱の差し色（黒猫飯店の朱ネオン看板）。
  `docs/DESIGN_SYSTEM.md` のトークンを土台に、モックアップの暖色に寄せる。

### 3.1 潜行（DIVE）
- ダンジョン SubViewport を大きく。HP/EP バー、ミニマップ、**大きなポモドーロ残時間（18:22/25:00）**、
  ダメージのフローティング数字、パーティのちびスプライト列、戦闘ログ。
- 既存の `dive_panel` / `timer_box` を再スタイル（構造は流用）。

### 3.2 帰還（CLOSE）カード
- 「今夜の首尾」：収穫（乾物/肉/海鮮）・到達階（15F→17F）・箱数。既存 close カードのリスキン。
- CTA「暖簾を出す」で営業中（NIGHT）へ。

### 3.3 営業中（NIGHT）＝刷新の主役
モックアップ画像3右上の「黒猫飯店・営業中」がこれ。現状の素っ気ない「閉店後」タブ羅列を置換。
- **ピクセル飯店の内観**（背景）＋カウンター/卓に座る仲間・常連のスプライト。
- **本日の献立**カード（料理3種＋個数バッジ）。
- **依頼パネル**：ヒロインのポートレート＋2択（例「赤い猫を探して／泣いてるテレビを探して」）。
- 箱開封・会話導線をこの画面に集約（休憩5分に収まる報酬パッケージ）。

### 3.4 依頼／会話オーバーレイ（ノベル風）
- 画像3左下のようにポートレート＋テキストボックス＋2択。既存 TalkScene を流用しリスキン。

### 3.5 タブ再編
- モックアップの下部：**献立 / 探索 / メンバー / アイテム /（SNS）/ 設定**。
  現状の「準備・倉庫・改装・統計」を上記にマッピング（準備→献立、倉庫→アイテム 等）。

### 実装フェーズ（小さく出す）
1. **P0：サイクル是正（§2）** — 文言＋ビュー切替のみ。低リスク・即体感が変わる。最優先。
2. **P1：共通トップバー＋パレット** — 全画面の見え方を底上げ。
3. **P2：営業中(NIGHT)画面の刷新（§3.3）** — 飯店内観＋献立＋依頼。
4. **P3：潜行/依頼オーバーレイのリスキン（§3.1, §3.4）**、タブ再編（§3.5）。

各フェーズで `godot --headless -s tests/test_sim.gd`（既存テスト）が緑であることを確認。
