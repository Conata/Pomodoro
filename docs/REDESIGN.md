## 1. 違和感の正体：「ポモドーロ後に閉店」問題（起点）

現状 `MORNING(開店前) → DIVE(潜行) → CLOSE(精算・閉店三行) → NIGHT("閉店後") → 翌朝`。
集中を完走した直後に「精算」→「閉店後」が出る＝**休憩がまるごう"店じまい後の事務処理"**。
集中のご褒美が「閉店レシート」になっていて逆。→ v2 の3状態モデルで根本解消する。

## 2. 新サイクル：3つの状態（設計ボード準拠）

時間の使われ方で状態が決まる。**「店が開く＝接客」は休憩ではなく"放置（非集中）"の状態に置く。**

| 状態 | 実時間トリガ | 中身 | 店 |
|---|---|---|---|
| **A 探索** | ポモドーロ中（25分集中） | ダンジョン自動探索・戦闘・採取（介入しない） | 閉（皆が潜行中） |
| **B キャンプ** | 休憩中（5〜10分） | ダンジョン内で焚き火休憩。HP/EP回復・会話・装備/献立の支度・戦利品整理 | 閉 |
| **C お店モード** | ポモドーロをしていない放置時 | **暖簾を出す → 客が来る接客シミュ（後述）**。放置インカム・物語イベント | **開** |

### 流れ（1サイクル）
```
[C お店モード] ──「集中を始める/潜る」──▶ [A 探索 25分] ──完走──▶ [B キャンプ 休憩]
     ▲                                                                  │
     └────────────「セッションを終える/浮上」───────────────────────────┘
                         （続けるなら B→A で次階へ深く潜る）
```
- 集中を続ける限り A↔B を往復して深く潜る（休憩はダンジョン内キャンプ）。
- 集中をやめて戻ると **C お店モード**：店を開けて接客。ここが日常の「居場所」かつ放置収益源。

### 現 enum との対応（セーブ互換のため値は据え置き）
- `DIVE` → **A 探索**（ほぼ現状）。
- `NIGHT`/`MORNING`/`CLOSE` の「閉店」枠 → **C お店モード**（接客シミュ）へ再定義。
- 新規 **B キャンプ**：休憩フェーズとして追加（enum に `CAMP` を足すか、DIVE のサブ状態にするかは実装時判断）。
- `close_day()` の計算式は **C の収益ロジックの素地として再利用**（捨てない）。

## 3. お店モード（接客シミュ）＝デイブ・ザ・ダイバー的

ユーザー要望の核。**「店を開ける」と客が実時間で来店し始めるライブ・シミュ**。放置（非集中）時の遊び。

### ループ
1. **暖簾を出す（開店）**：献立（探索で得た食材/レシピ）をセットして開店。
2. **来店**：客が時間経過で到着（到着レートは看板/評判・時間帯で変動。ポアソン的）。
3. **注文 → 提供**：客は献立から注文。提供（自動 or タップ）。**味マッチで単価1.2倍**（既存 `close_day` の係数を流用）。在庫（食材）を消費。
4. **会計 → 評判**：支払い＝収益、満足＝評判（看板）上昇 → 次の来店レート増。待たせ過ぎ＝離脱。
5. **物語フック**：特定の客＝会話イベント発火（既存 `talk`/`event_data` を接続）。常連の常連化。
6. **放置収益**：完全離席中はオフライン精算（既存「安息」枠を"店が回り続けた"演出に転用。上限あり）。

### 既存資産の再利用
- `sim.close_day()`：客数=(8+看板)×フラグ、味マッチ単価1.2倍、店番適性×好感度→仕込み力。
  → **一括精算からライブ進行へ**作り替えるが、係数・式は土台に。
- 食材/在庫/献立デッキ：探索の戦利品が接客の弾になる（A→C の経済接続。設計ボードの「探索で得た食材で店を回す」と一致）。
- 店番シナジー5種（ご当地フェア/フルコース/店内ライブ/解析仕込み/静かな給仕）＝接客中の特殊効果へ。

### 接客シミュの実装メモ（純ロジック＝テスト可能に）
- `src/sim/shop.gd`（新規・エンジン非依存）：開店state・客キュー・到着RNG（既存 `rng.gd` のシード付き）・
  提供/会計/評判の遷移を純関数で。`tests/` にユニットテスト追加（GUT観点）。
- ビューは `src/ui/shop_view.gd`（新規）：飯店内観＋席＋来店アニメ＋注文吹き出し＋会計演出。
- 時間進行は既存ダイブと同じ **Date.now()アンカー＋固定ステップのキャッチアップ**を流用（離席中も正確）。

## 4. キャンプ（休憩・ダンジョン内）

休憩5〜10分は「店」ではなく**ダンジョン内の焚き火キャンプ**（設計ボード②）。
- HP/EP回復、パーティ会話（既存 banter/talk）、装備・献立の支度、戦利品/箱の整理。
- 次階へ潜る前の"溜め"。短い休憩に収まる報酬パッケージ（DESIGN.md の思想を踏襲）。

## 5. 画面刷新（UIは"額縁"。主役はキャラと店）

### 思想（最重要）
ゲームUIではなく **「ずっと開いていたくなるアプリ」**。近いのは
**深夜の喫茶店 × ノートアプリ × スクリーンセーバー**。主役は HPバーではなく**キャラクター**。
UIは情報を見せる箱ではなく、**キャラと店を引き立てる額縁**。

UIの優先順位：①常時表示しても疲れない ②キャラが見える ③作業の邪魔にならない ④休憩中だけ情報量が増える。

### モード別の情報密度（3軸カラーで現在地が一目で分かる）
識別色：**オレンジ=店 / ミント=ポモドーロ / 紫=キリコ・精神世界**。

- **C お店モード（非ポモドーロ＝ホーム画面・最重要）**：画面の主役は賑わう飯店。UIは額縁。
  近いのは **あつ森 / ねこあつめ**。接客シミュ（`ShopSim`）は**この画面にいる間ずっと裏で稼働**（客が来て会計＝放置収益）。構成（参照モック準拠）：
  - 上部バー：黒猫飯店ロゴ＋時刻/日付（例 23:41 5/24）＋**集中タイマー 25:00**＋通知ベル＋メニュー。
  - 左ナビ：店モード / 探索モード / 記憶の欠片 / 設定（現在地を橙で点灯）。
  - 中央：ネオン中華の店内（赤ネオン看板・カウンター・キャラ・黒猫・夜の青い窓）。
  - 下：大CTA **「集中を始める」＝ミント（ポモドーロ）**＋「長押しでクイック開始」。
  - 右下スタッツ：評判 ★★★☆☆ 3.2 ／ 資金 12,840G ／ 記憶の欠片 128（sim の sign/gold/shards に対応）。
  - ※開始ボタンは「暖簾を出す(橙)」ではなく「集中を始める(ミント)」。店は常に開いており、押すと探索(A)へ。
- **B キャンプモード（休憩）**：情報量を増やす。焚き火＋パーティ（ミル/ムュウ/レイカ/フズキ）＋会話ログ。
  近いのは **ダクソの篝火 / Stellar Blade のキャンプ**。
- **A 探索モード（ポモドーロ中）＝戦闘画面**：放置のオートバトルを**映像として見せる**（スクリーンセーバー的に
  眺められる）。**テキストの戦闘ログは出さない**。構成（横画面の参照モック準拠）：
  - 上部バー：黒猫飯店ロゴ＋現在地（例「忘却区画・B2F」）＋**ポモドーロ 18:27/25:00＋進捗バー**＋ジェム/ゴールド。
  - 左パネル：探索率(例72%)・現在の出来事・**遭遇『人格・後悔』**・ミニマップ。
  - 中央：サイドビューの自動戦闘（パーティ4人＋紫オカルト系の敵＋ポータル、HP♥）。色はダンジョン
    通常`#2B2D42`／オカルト`#5B3A82`／ボス`#A63D40`。既存 `src/ui/dive_view.gd` を土台に拡張。
  - 下部：メンバーカード4（ミル/ムュウ/レイカ/フズキ）＋操作アイコン（探す/アイテム/設定）。
  - 向き：**PC版＝横画面（この参照が正）／モバイル版＝縦に再構成**（上=情報・中=戦闘・下=メンバー）。下記フォームファクタ方針参照。

共通トップバー（時刻＋場所＋所持金）は薄く。タブ再編：献立 / 探索 / メンバー / アイテム /（SNS）/ 設定。

### 対応フォームファクタ（PC＝横／モバイル＝縦の2系統）
本作は **PC版（横画面）とモバイル版（縦画面）の両対応**。提供リファレンス（戦闘画面・店モード）は
**PC版＝横画面が正**。モバイル版は同じ内容を縦に再構成する。

- **共通（向き非依存・使い回す）**：ゲームロジック（`sim.gd`/`shop.gd`）、UIキット9-patch、パレット、
  キャラ/演出素材。＝既に実装済みの部分はそのまま両対応で効く。
- **分岐（向きで変える）**：各モードの**レイアウトのみ**。実装はアスペクト比で判定し、
  `_layout_landscape()` / `_layout_portrait()` のように画面構成を切替（同じノード/データを別配置）。
- モック：縦＝送付済み3枚（モバイル）。横＝PC版はリファレンス画像が当面の正。必要なら横モックも生成する。
- project.godot は現在 縦720×1280。PC版対応時は最小幅の確保とリサイズ追従（stretch=canvas_items/expand は既に可）。

### 素材調達の制約（重要）
- **この実行環境は egress 制限があり、Kenney/itch.io/0x72 等から UI キット原本をダウンロードできない**
  （調査時すべて 403）。実物を入れるには **(a) 環境の egress 許可リストに配布元を追加**してもらうか、
  **(b) こちらで生成したナインスライスUIキット（音声と同じく依存ゼロ生成）を暫定採用**し、

  共通化しておけば差し替えはテクスチャ入替だけで済む。
- 採用素材は `assets/third_party/CREDITS.md` 形式でクレジット管理（CC0優先）。
